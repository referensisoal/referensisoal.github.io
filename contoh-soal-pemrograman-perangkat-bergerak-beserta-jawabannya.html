pekerjaan manusia. Beberapa unsur utama dalam membuat sistem robot adalah membangun

peralatan mekanik, perangkat keras dan perangkat lunak. Keterbatasan biaya dalam membuat

perangkat keras robot dan kemampuan pemrograman merupakan hambatan utama dalam

mempelajari dan mengembangkan robot. Berkembangnya komunitas robot pada saat ini menjadikan

Robot Operating System (ROS) dan Gazebo sebagai media interaktif untuk belajar dan membuat

simulasi robot. ROS merupakan sebuah middleware yang dilengkapi dengan tools dan library untuk

membuat perangkat lunak robot, sedangkan Gazebo merupakan sebuah aplikasi simulasi 3D yang

dapat digunakan untuk membangun desain perangkat keras robot. Dalam penulisan ini, peneliti

membuat sebuah simulasi robot penghindar benda menggunakan sensor lidar dan kamera, simulasi

Kata Kunci: Robot Operating System (ROS), Gazebo, Robot.

Abstract

A robot is a prototype that is able to control by automatically to help the human work. Some of

the parts to build a robot system are making the mechanical system, hardware, and software. The

limitation of cost to make the robot hardware and programming skills are an obstacle to learn and

develop the robot system. The development of the robot community nowadays makes the Robot

Operating System (ROS) and Gazebo as an interactive media to learn and make the robot simulation.

ROS is a middleware that equipped with tools and libraries to create a robot software, while Gazebo is

a 3D simulation application to build a robot hardware design. On this paper, the researcher makes a

robot simulation to avoid the object using Lidar sensor and camera, the simulation has developed

using ROS and Gazebo that be able to use as a media for robot learning interactive.

Keyword: Robot Operating System (ROS), Gazebo, Robot.

28. Sebutkan 6 contoh bahasa pemrograman beserta kelebihan atau kekurangan​


Berikut adalah 6 contoh bahasa pemrograman beserta beberapa kelebihan dan kekurangan masing-masing:

1. **Python**:

- Kelebihan:

- Mudah dibaca dan ditulis, sehingga cocok untuk pemula.

- Memiliki berbagai pustaka dan framework untuk berbagai aplikasi.

- Dukungan komunitas yang besar.

- Kekurangan:

- Tidak secepat bahasa pemrograman lain seperti C atau C++.

- Gudang Pustaka yang sangat besar bisa membingungkan pemula.

2. **Java**:

- Kelebihan:

- Portabilitas tinggi karena menggunakan mesin virtual Java (JVM).

- Berkeamanan tinggi dengan kontrol akses ketat.

- Populer di pengembangan perangkat lunak berbasis web dan perangkat seluler.

- Kekurangan:

- Membutuhkan lebih banyak kode dibanding bahasa lain untuk operasi sederhana.

- Kinerja sedikit lebih lambat dibanding bahasa yang dikompilasi.

3. **C++**:

- Kelebihan:

- Mendukung pemrograman berorientasi objek.

- Kinerja tinggi dan dapat diandalkan.

- Memungkinkan pengelolaan memori langsung.

- Kekurangan:

- Bahasa yang lebih rumit, membuatnya sulit untuk dipahami oleh pemula.

- Potensial untuk kesalahan memori (segmen ganda, kebocoran memori) jika tidak dielola dengan baik.

4. **JavaScript**:

- Kelebihan:

- Digunakan untuk pengembangan web dan dapat dijalankan di semua peramban.

- Kemampuan pemrograman berorientasi objek.

- Dukungan komunitas yang kuat.

- Kekurangan:

- Memiliki beberapa keanehan bahasa yang dapat menyebabkan bug jika tidak hati-hati.

- Bergantung pada peramban untuk menjalankan kode, yang dapat menghasilkan perilaku yang berbeda.

5. **Ruby**:

- Kelebihan:

- Mudah dibaca dan menulis dengan sintaks yang bersahabat.

- Dukungan kuat untuk pengembangan web (Ruby on Rails).

- Fokus pada produktivitas pengembang.

- Kekurangan:

- Kinerja relatif lebih lambat dibanding bahasa seperti C++.

- Dukungan komunitas yang lebih kecil dibandingkan bahasa pemrograman yang lebih populer.

6. **Rust**:

- Kelebihan:

- Keamanan tinggi dengan sistem peminjaman dan pengecekan memori yang ketat.

- Kinerja tinggi.

- Dukungan untuk pemrograman konkuren dan paralel.

- Kekurangan:

- Memiliki kurva pembelajaran yang lebih curam, terutama dalam hal sistem peminjaman.

- Belum sepopuler bahasa lain seperti Python atau Java.

Setiap bahasa pemrograman memiliki kelebihan dan kekurangan yang harus dipertimbangkan sesuai dengan kebutuhan proyek Anda. Pemilihan bahasa pemrograman tergantung pada jenis aplikasi yang akan Anda kembangkan, tingkat pengalaman Anda, dan preferensi Anda dalam hal sintaks dan fitur.
